# CodeGen Overview: SimpleEcho

> Guides the code generator when producing Swift files for this agent.

---

## Purpose

This `specs/` directory holds agent-specific generation specs that supplement the shared rules in `CodeGenSpecs/`. The generator reads both sets of specs and merges them when producing the output.

---

## Agent Module Structure

The generator will produce the following Swift files for this agent:

| File | Purpose |
|------|---------|
| `SimpleEcho.swift` | Main `@SpecDrivenAgent` actor with `execute(goal:)` |
| `SimpleEchoCLI.swift` | CLI runner using swift-argument-parser |
| `SimpleEchoTests.swift` | Swift Testing suite |

No additional files are needed — this agent has no tools, no background execution, and no transcript extensions.

---

## Customization Points

### State Properties
Uses macro-generated defaults only (`_status: AgentStatus`, `_transcript: ObservableTranscript`, `_client: LLMClient?`). No additional properties.

### Tool Dispatch
None. This agent does not register or dispatch any tools.

### Background Checkpoint Format
None. This agent does not support background execution.

### SwiftUI Integration Notes
Not applicable for this proof-of-concept agent.

---

## Notes

- `execute(goal:)` is the agent-specific method returning `String`; the macro-generated `run(goal:)` (void return, delegates to `AgentRuntime`) coexists but is not used by this agent
- Accesses `_status` and `_transcript` directly (private members generated by macro, accessible within the same actor)
- Uses `AgentStatus.error(Error)` for failures and `AgentStatus.completed(Any)` for success
- No `client` injection needed since this agent never calls an LLM
