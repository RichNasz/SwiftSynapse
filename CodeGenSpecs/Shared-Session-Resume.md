# Shared Spec: Session Resume

> Defines `AgentSession` — a codable snapshot of agent state — and the `resume(from:)` entry point that allows agents to continue a previous execution without restarting from scratch.

---

## Summary

Long-running agents (PRReviewer, ResearchAssistant, BackgroundAgent) may be interrupted by app termination, system resource pressure, or background expiration. `AgentSession` captures the in-flight state of an agent. `resume(from:)` replays that state and continues execution from the last incomplete step.

`ObservableTranscript` already holds all conversation entries. `AgentSession` wraps the transcript plus minimal additional context needed to re-enter `execute()` at the right point.

---

## AgentSession Type

```swift
public struct AgentSession: Codable, Sendable {
    /// The Swift type name of the actor (e.g., "LLMChat", "PRReviewer").
    public let agentType: String

    /// The original goal string passed to execute().
    public let goal: String

    /// The transcript at the time of interruption — all entries appended so far.
    public let transcriptEntries: [TranscriptEntry]

    /// Index of the last successfully completed step.
    /// Agents use this to skip already-done work when resuming.
    public let completedStepIndex: Int

    /// Optional agent-specific serialized state (custom stored properties).
    /// Encoded by the agent using JSONEncoder; decoded by the agent at resume time.
    public let customState: Data?

    /// Wall-clock time when the session was created.
    public let createdAt: Date

    /// Wall-clock time when the session was last saved.
    public let savedAt: Date
}
```

`TranscriptEntry` must be `Codable` for this to work. If `SwiftOpenResponsesDSL.TranscriptEntry` is not yet `Codable`, agents use a local `CodableTranscriptEntry` mirror that is `Codable` and converts to/from `TranscriptEntry`.

---

## Session Persistence (Out of Scope)

`AgentSession` defines the **in-memory contract** only. Where and how a session is persisted (disk, iCloud, Core Data, `UserDefaults`) is the caller's responsibility. Agents create and consume `AgentSession` values; they do not write to storage.

Typical caller pattern:
```swift
// Before backgrounding
let session = await agent.currentSession()
try JSONEncoder().encode(session).write(to: sessionURL)

// On next launch
let session = try JSONDecoder().decode(AgentSession.self, from: Data(contentsOf: sessionURL))
try await agent.resume(from: session)
```

## resume(from:) Entry Point

`resume(from:)` is generated by `@SpecDrivenAgent` for agents whose spec declares `supportsResume: true`:

```swift
// Macro-generated signature — return type matches the agent's execute() return type
public func resume(from session: AgentSession) async throws -> String
```

Note: All current agents return `String` from `execute()`. If a future agent returns a different type, the `resume(from:)` signature should match that type. The `AgentStatus.completed(Any)` uses type erasure; the `resume()` return type is the concrete type, not `Any`.

Behavior:
1. Validate `session.agentType == String(describing: Self.self)` — throw `AgentError.sessionTypeMismatch` if mismatched.
2. Restore `_transcript` from `session.transcriptEntries` (via `_transcript.restore(entries:)`).
3. Restore `customState` if present (agent-specific decode).
4. Call `execute(goal: session.goal, resumingFrom: session.completedStepIndex)`.

The `resumingFrom` parameter is an internal overload of `execute()` that skips steps `0..<completedStepIndex`. External callers always use `resume(from:)` — never `execute(goal:resumingFrom:)` directly.

---

## Step Index Convention

Agents that support resume must number their execution steps. Steps are numbered starting at 0. The agent records `completedStepIndex` after each step completes:

```swift
// In execute():
// Step 0: validate and set running
_status = .running
_transcript.append(.userMessage(goal))
_agentContext.completedStepIndex = 0

// Step 1: first LLM call
let response = try await retryWithBackoff(...) { try await _llmClient.send(firstRequest) }
_agentContext.completedStepIndex = 1

// Step 2: optional persona rewrite
if let persona {
    let rewrite = try await retryWithBackoff(...) { try await _llmClient.send(secondRequest) }
    _agentContext.completedStepIndex = 2
}
```

`_agentContext.completedStepIndex` is serialized into `AgentSession.completedStepIndex` when `currentSession()` is called.

## currentSession() Method

Agents expose a method to snapshot the current state:

```swift
// Macro-generated for agents with supportsResume: true
public func currentSession() -> AgentSession
```

Called by the background execution layer (see `Shared-Background-Execution.md`) at checkpoint moments, and by the host app before saving to disk.

## Transcript Restoration

`ObservableTranscript` gains a `restore(entries:)` method (generated alongside the session resume feature):

```swift
// In ObservableTranscript
public func restore(entries: [TranscriptEntry]) {
    self.entries = entries
    self.isStreaming = false
    self.streamingText = ""
}
```

This is distinct from `reset()` (which clears to empty) and `append(_:)` (which adds one entry).

---

## Validation and Error Cases

Required additions to agent error enums when `supportsResume: true`:

```swift
case sessionTypeMismatch   // session.agentType does not match this actor type
case sessionCorrupted      // customState could not be decoded
```

These are terminal errors — a corrupted session is not retried.
